# Types

TypeScript type definitions for Motion.

## Transition

A `Transition` is an object that defines how values animate from one state to another.

<div>

This object can contain [`Orchestration` props](#orchestration) like `delay`, that schedule the animation as a whole.

</div>

```jsx
<motion.div
  animate={{ x: 100 }}
  transition={{ delay: 1 }}
/>
```

<div>

It can also accept props that define which type of animation to use a `Tween`, `Spring` or `Inertia`.

</div>

```jsx
<motion.div
  animate={{ x: 100 }}
  transition={{ type: "spring", stiffness: 100 }}
/>
```

<div>

A different type of animation can be defined for each animating value. `default` can be used to define animations for any remaining values.

</div>

```jsx
<motion.div
  animate={{ x: 100, y: 100, opacity: 1 }}
  transition={{
    delay: 1,
    x: { type: "spring", stiffness: 100 },
    default: { duration: 2 },
  }}
/>
```

## Spring

An animation that simulates spring physics for realistic motion.

This is the default animation for physical values like `x`, `y`, `scale` and `rotate`.

## Inertia

An animation that decelerates a value based on its initial velocity,
usually used to implement inertial scrolling.

Optionally, `min` and `max` boundaries can be defined, and inertia
will snap to these with a spring animation.

This animation will automatically precalculate a target value,
which can be modified with the `modifyTarget` property.

This allows you to add snap-to-grid or similar functionality.

Inertia is also the animation used for `dragTransition`, and can be configured via that prop.
